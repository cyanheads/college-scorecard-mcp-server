# 0.1.6 — 2026-06-19

## Changed

- **`repayment_rate_3yr` → `repayment_progress_3yr`** on `scorecard_get_school`, `scorecard_value_analysis`, and `scorecard_compare_schools` ([#6](https://github.com/cyanheads/college-scorecard-mcp-server/issues/6)). The old field read `latest.repayment.3_yr_repayment.overall` — a borrower **count** (e.g. `10786` for UW) divided by 1000 and rendered as a percentage, producing impossible values like 1078.6%. It now reads `latest.repayment.repayment_cohort.3_year_declining_balance`, the API's 0–1 fraction. The College Scorecard API exposes no overall 3-year *repayment rate*; declining balance is the share of the cohort paying down principal, so the surfaced field is labeled repayment **progress** (declining balance), not a rate. Text labels read "3-Year Repayment Progress (declining balance)".
- **Dependencies** — `@biomejs/biome` `^2.4.16` → `^2.5.0`, `@types/node` `^25.9.3` → `^25.9.4`, `vitest` `^4.1.8` → `^4.1.9`.

## Fixed

- **Net price by income bracket** now populates on `scorecard_get_school`, `scorecard_value_analysis`, and `scorecard_compare_schools` ([#7](https://github.com/cyanheads/college-scorecard-mcp-server/issues/7)). The previous `latest.cost.avg_net_price.by_income.*` paths do not exist — the API silently dropped them, so every bracket returned null. Each tool now requests both `latest.cost.net_price.public.by_income_level.*` and `latest.cost.net_price.private.by_income_level.*` and coalesces public → private per bracket, resolving a real value whichever ownership the school has. UW (public, $25k income) now returns `net_price_for_income: 6384`; Harvard (private) resolves from the private path.
