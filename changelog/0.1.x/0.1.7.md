# 0.1.7 — 2026-06-19

## Changed

- **`@cyanheads/mcp-ts-core` `^0.10.6` → `^0.10.8`** — framework maintenance pickup. Tool, resource, and prompt surface unchanged. Relevant deltas:
  - `devcheck` no longer false-fails on a fresh, pre-`git init` scaffold: the TODOs/FIXMEs, Tracked Secrets, and Framework Antipatterns checks share an `isGitRepo()` guard, and `build-changelog.ts --check` exits cleanly when only `changelog/template.md` is present ([mcp-ts-core #242](https://github.com/cyanheads/mcp-ts-core/issues/242), [#243](https://github.com/cyanheads/mcp-ts-core/issues/243)).
  - `check-skill-versions.ts` skips a `SKILL.md` deleted from the worktree instead of crashing with ENOENT, so a maintenance pass that prunes an upstream-removed skill no longer breaks ([mcp-ts-core #237](https://github.com/cyanheads/mcp-ts-core/issues/237)).
  - `biome.json` migrated to the biome `2.5.0` schema (`recommended` → `"preset": "recommended"`); inherited via `@cyanheads/mcp-ts-core/biome`.
  - `DuckdbProvider.describe()` filter qualification and the canvas SQL-gate `invalid_sql` prepare-failure classification ([mcp-ts-core #235](https://github.com/cyanheads/mcp-ts-core/issues/235), [#236](https://github.com/cyanheads/mcp-ts-core/issues/236)) — framework-internal; this server exposes no DataCanvas/SQL surface.
- **Re-synced project skills** — seven `skills/<name>/SKILL.md` `metadata.version` bumps propagate upstream body drift ([mcp-ts-core #238](https://github.com/cyanheads/mcp-ts-core/issues/238)): `api-auth`, `api-errors`, `api-services`, `api-telemetry`, `field-test`, `report-issue-local`, `tool-defs-analysis`. `git-wrapup` (v1.4) and `orchestrations` (v1.4) reworded; `add-tool`, `api-canvas`, `api-config`, `api-context` bodies refreshed. Scripts (`build-changelog.ts`, `check-framework-antipatterns.ts`, `check-skill-versions.ts`, `devcheck.ts`) re-synced from the framework.
- **`devcheck.config.json`** — emptied `outdated.allowlist` (`@cyanheads/mcp-ts-core`, `@types/node` no longer held back) and `skillVersions.ignore` (`report-issue-local`), both now in sync.

### Dependency bumps

- `@cyanheads/mcp-ts-core` `^0.10.6` → `^0.10.8`
- `@types/node` `^25.9.4` → `^26.0.0`
