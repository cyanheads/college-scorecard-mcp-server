# 0.1.5 — 2026-06-12

## Added

- **`ctx.enrich` out-of-band metadata** — `scorecard_search_schools`, `scorecard_search_programs`, `scorecard_list_fields`, and `scorecard_lookup_cip` emit a `truncated` signal (`shown`, `cap`, and a guidance hint) when a page or list fills to its limit; the two search tools also emit `total` — institutions/schools matching before pagination.
- **`name` / `title` on `createApp()`** — set explicitly to `college-scorecard-mcp-server` so every client surface shows the hyphenated repo name.
- **`.claude-plugin/plugin.json`, `.codex-plugin/plugin.json` + `mcp.json`** — Claude Code and Codex plugin manifests for marketplace install.
- **`CITATION.cff`** — software citation metadata.
- **Dockerfile `HEALTHCHECK`** — probes `/healthz` every 30s; `org.opencontainers.image.version` label stamped from the `APP_VERSION` build arg.
- **`scripts/release-github.ts`, `scripts/clean-mcpb.ts`, `scripts/check-skill-versions.ts`** — release-github helper, `.mcpb` sanitizer (wired into `bundle`), and a devcheck skill-version-drift check.
- **`format:unsafe` script** — `format` now runs Biome without `--unsafe`; `format:unsafe` keeps the unsafe pass.

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.9` → `^0.10.6` — framework adoption; drove the enrichment migration below.
- **`notice` moved from `output` to `enrichment`** on `scorecard_compare_schools`, `scorecard_get_school`, `scorecard_get_programs`, `scorecard_search_schools`, and `scorecard_search_programs` — the warning is no longer a structured-content field, and `format()` text no longer renders the notice line; `ctx.enrich.notice` carries it.
- **`devcheck`** — added Open-Indexed Interfaces and Skill Versions checks; skills-sync now flags stale framework skills (mirror-only dirs marked `audience: external`).
- **`.mcpbignore`** — anchored ignore globs to repo root (`/`-prefixed); excludes `.agents/` and `skills/` from the bundle.
- **README MCP client keys** — example configs key the server as `college-scorecard-mcp-server` (was `college-scorecard`).
- **Tests** — adopt `getEnrichment` from `@cyanheads/mcp-ts-core/testing` to assert enrichment signals.
- **Dependencies** — `@biomejs/biome` `^2.4.15` → `^2.4.16`, `@types/node` `^25.9.1` → `^25.9.3`, `vitest` `^4.1.7` → `^4.1.8`.

## Removed

- **`skills/migrate-mcp-ts-template`** — removed upstream from the framework template.
